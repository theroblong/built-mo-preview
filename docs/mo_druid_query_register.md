# Mo Druid Query Register

This register contains the actual Druid SQL blocks behind the Mo ML playbook. Query IDs in the playbook link here so an engineer or analyst can jump directly from the play stage to the SQL to review, copy, parameterize, or test in Druid.

## Datasource Configuration

- Current raw SPINS datasource: `spins_full`.
- If the raw datasource name changes, update Q0 before running the materialization sequence.
- Q0 is the only query below that reads the raw uploaded datasource directly. Later queries read governed intermediate tables such as `built_filtered_weekly`, `built_enriched_weekly`, and feature tables.
- Parameterized UI queries use placeholders such as `:focal_upc`; replace those with test values in the Druid SQL console or bind them from the application layer.

## Q2 Universal comparison pool

**Purpose:** Self-join to build every valid (focal_upc, candidate_upc) pair. `comparison_type` and `relationship_distance` are internal columns — UI maps them to plain-English labels at render time.

**Context required:** Set `{"sqlJoinAlgorithm": "sortMerge"}` in the Druid SQL console context panel before running — Q2 self-joins `built_enriched_weekly` which exceeds the cluster's broadcast join memory limit (~311 MB). sortMerge streams both sides instead of broadcasting.

```sql

SET sqlSortMergeDiskBuffered = 'true';              -- spills sort-merge merge buffers to disk; reduces per-worker memory pressure
SET durableShuffleStorage = 'true';               -- routes shuffle files to S3; cluster-level config completed by Rob

REPLACE INTO "comparison_pool_weekly"
OVERWRITE WHERE __time >= TIMESTAMP '2025-01-01'

  f.geography_raw, f.geography_display, f.geography_level,
  f.channel_outlet, f.channel,
  f.retail_account, f.retail_account_level,

  -- Focal item
  f.upc                                                  AS focal_upc,
  f.parent_brand                                         AS focal_brand,
  f.brand_line                                           AS focal_brand_line,
  f.description                                          AS focal_description,
  f.specific_flavor_normalized                           AS focal_flavor,
  f.spins_flavor                                         AS focal_spins_flavor,

  f.size_oz                                              AS focal_size_oz,
  f.arp                                                  AS focal_arp,
  f.base_arp                                             AS focal_base_arp,
  f.units_pct_promo                                      AS focal_units_pct_promo,
  f.promo_weeks                                          AS focal_promo_weeks,
  f.first_week_selling                                   AS focal_first_week_selling,

  -- Candidate item
  c.upc                                                  AS candidate_upc,
  c.parent_brand                                         AS candidate_brand,
  c.brand_line                                           AS candidate_brand_line,
  c.description                                          AS candidate_description,
  c.specific_flavor_normalized                           AS candidate_flavor,
  c.spins_flavor                                         AS candidate_spins_flavor,
  c.pack_count                                           AS candidate_pack_count,
  c.size_oz                                              AS candidate_size_oz,

  -- Internal comparison type (backend/ML only)
  CASE
    WHEN f.specific_flavor_normalized = c.specific_flavor_normalized
     AND f.parent_brand = c.parent_brand
     AND f.pack_count  != c.pack_count
      THEN 'SAME_SPECIFIC_FLAVOR_SAME_BRAND_PACK_LADDER'
    WHEN f.specific_flavor_normalized = c.specific_flavor_normalized
     AND f.parent_brand != c.parent_brand
      THEN 'SAME_SPECIFIC_FLAVOR_CROSS_BRAND'
    WHEN f.spins_flavor = c.spins_flavor
     AND f.parent_brand = c.parent_brand
     AND f.specific_flavor_normalized != c.specific_flavor_normalized
      THEN 'SAME_FLAVOR_SAME_BRAND'
    WHEN f.spins_flavor = c.spins_flavor
     AND f.parent_brand != c.parent_brand
      THEN 'SAME_FLAVOR_CROSS_BRAND'
    WHEN f.spins_flavor != c.spins_flavor
     AND f.parent_brand = c.parent_brand
      THEN 'CROSS_FLAVOR_SAME_BRAND'
    ELSE 'CROSS_FLAVOR_CROSS_BRAND'
  END AS comparison_type,

  -- Internal relationship distance (ML feature; never shown as a number in UI)
  CASE
    WHEN f.specific_flavor_normalized = c.specific_flavor_normalized
     AND f.parent_brand = c.parent_brand
     AND f.pack_count  != c.pack_count                   THEN 1
    WHEN f.specific_flavor_normalized = c.specific_flavor_normalized
     AND f.parent_brand != c.parent_brand                THEN 2
    WHEN f.spins_flavor = c.spins_flavor
     AND f.parent_brand = c.parent_brand
     AND f.specific_flavor_normalized != c.specific_flavor_normalized
                                                         THEN 3
    WHEN f.spins_flavor = c.spins_flavor
     AND f.parent_brand != c.parent_brand                THEN 4
    WHEN f.spins_flavor != c.spins_flavor
     AND f.parent_brand = c.parent_brand                 THEN 5
    ELSE 6
  END AS relationship_distance,

    WHEN f.parent_brand = c.parent_brand
     AND f.specific_flavor_normalized = c.specific_flavor_normalized

    ELSE NULL
  END AS pack_distance,

    WHEN c.pack_count > 0 AND f.pack_count > 0

  c.store_count                                          AS candidate_store_count,
  c.stores_selling                                       AS candidate_stores_selling,
  c.pct_stores_selling                                   AS candidate_pct_stores_selling,
  c.tdp_any_promo                                        AS candidate_tdp_any_promo,
  c.tdp_non_promo                                        AS candidate_tdp_non_promo,

  c.units_lift_any_display                               AS candidate_units_lift_any_display,
  c.units_lift_any_feature                               AS candidate_units_lift_any_feature,

    WHEN 'PERFECT BAR'               THEN 1

    WHEN 'NUGO NUTRITION'            THEN 2

    WHEN 'CLIF BAR'                  THEN 3
    WHEN 'LARABAR'                   THEN 3
    WHEN 'BOBOS'                     THEN 3
```

## Q2c Cross-flavor heatmap aggregation

**Purpose:** Powers the geography × SKU heatmap on the Cross-flavor screen. Returns `base_units_pct_chg` and a `heatmap_bucket` for each (SKU, geography) cell. Run at render time against `built_prepost_features` — not pre-materialized.

Five heatmap buckets match CSS class names: `hm-strong-neg` (< −10%), `hm-mild-neg` (−10% to −3%), `hm-neutral` (−3% to +3%), `hm-mild-pos` (+3% to +10%), `hm-strong-pos` (> +10%).

```sql
-- Parameters: :spins_flavor, :channel_outlet, :retail_account
SELECT
  f.upc,
  f.description,
  f.pack_count,
  f.geography_raw,
  f.geography_display,
  f.retail_account,
  f.channel_outlet,
  f.pre_13w_base_units,
  f.post_13w_base_units,
  -- Percentage change (13w default)
  (f.post_13w_base_units - f.pre_13w_base_units)
    / NULLIF(f.pre_13w_base_units, 0)              AS base_units_pct_chg,
  -- Heatmap bucket matching CSS class names in the UI
  CASE
    WHEN (f.post_13w_base_units - f.pre_13w_base_units)
           / NULLIF(f.pre_13w_base_units, 0) < -0.10  THEN 'hm-strong-neg'
    WHEN (f.post_13w_base_units - f.pre_13w_base_units)
           / NULLIF(f.pre_13w_base_units, 0) < -0.03  THEN 'hm-mild-neg'
    WHEN (f.post_13w_base_units - f.pre_13w_base_units)
           / NULLIF(f.pre_13w_base_units, 0) <= 0.03  THEN 'hm-neutral'
    WHEN (f.post_13w_base_units - f.pre_13w_base_units)
           / NULLIF(f.pre_13w_base_units, 0) <= 0.10  THEN 'hm-mild-pos'
    ELSE 'hm-strong-pos'
  END AS heatmap_bucket
FROM "built_prepost_features" f
JOIN "built_enriched_weekly" e
  ON  f.upc          = e.upc
  AND f.channel_outlet = e.channel_outlet
  AND f.retail_account = e.retail_account
  AND f.geography_raw  = e.geography_raw
WHERE
  e.spins_flavor      = :spins_flavor
  AND f.channel_outlet  = :channel_outlet
  AND f.retail_account  = :retail_account
  AND e.parent_brand    = 'BUILT'
  AND e.military_excluded_flag = 0
GROUP BY
  f.upc, f.description, f.pack_count,
  f.geography_raw, f.geography_display,
  f.retail_account, f.channel_outlet,
  f.pre_13w_base_units, f.post_13w_base_units
```

## Q2e Cannibalization rate trend retrieval

**Purpose:** Returns trailing actuals from `cannibalization_rate_weekly` plus future weeks from `cannibalization_rate_forecast_weekly` in one UI-ready result set. Powers the Forecast screen trend chart — actual weeks render as solid bars; forecast weeks render in the scenario color with a confidence band.

```sql
-- Parameters: :focal_upc, :candidate_pool_id, :scenario_id,
--             :channel_outlet, :retail_account, :geography_display,
--             :forecast_origin_week
SELECT
  __time,
  'ACTUAL'                                           AS series_type,
  cannibalization_rate,
  cannibalization_rate_low,
  cannibalization_rate_high,
  focal_units_gained,
  estimated_donor_units_lost,
  incremental_share
FROM "cannibalization_rate_weekly"
WHERE focal_upc         = :focal_upc
  AND candidate_pool_id = :candidate_pool_id
  AND channel_outlet    = :channel_outlet
  AND retail_account    = :retail_account
  AND geography_display = :geography_display
  AND __time >= TIMESTAMPADD(WEEK, -8, :forecast_origin_week)

UNION ALL

SELECT
  __time,
  'FORECAST'                                         AS series_type,
  cannibalization_rate_forecast                      AS cannibalization_rate,
  cannibalization_rate_forecast_low                  AS cannibalization_rate_low,
  cannibalization_rate_forecast_high                 AS cannibalization_rate_high,
  forecast_focal_units_gained                        AS focal_units_gained,
  forecast_donor_units_lost                          AS estimated_donor_units_lost,
  incremental_share_forecast                         AS incremental_share
FROM "cannibalization_rate_forecast_weekly"

  AND channel_outlet    = :channel_outlet
  AND retail_account    = :retail_account
  AND geography_display = :geography_display
  AND forecast_origin_week = :forecast_origin_week
```

## Q3 Focal pre/post window aggregation

**Purpose:** For each focal BUILT SKU, compute before/after metrics across all four window sizes (4w, 13w, 26w, YTD) in a single pass. Pre window = strictly before launch week. Post window = launch week and after. Keyed to `first_week_selling`.

```sql
REPLACE INTO "built_prepost_features"
OVERWRITE ALL
WITH weekly_tagged AS (
  SELECT
    w.__time,
    w.upc, w.description, w.channel_outlet, w.retail_account,
    w.geography_raw, w.geography_display, w.geography_level,
    w.parent_brand, w.brand_line,
    w.specific_flavor_normalized, w.spins_flavor, w.pack_count,
    w.base_units, w.base_units_yago,
    w.units, w.units_yago,
    w.tdp, w.tdp_yago,
    w.avg_weekly_units_spm,
    w.avg_weekly_units_per_store_selling_per_item,
    w.pct_stores_selling, w.stores_selling,
    w.arp, w.arp_yago, w.base_arp,
    w.promo_weeks, w.units_pct_promo, w.arp_pct_discount_any_promo,
    w.first_week_selling,
    TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd')       AS launch_date,
    TIMESTAMPDIFF(WEEK,
      TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd'),
      w.__time)                                          AS weeks_from_launch,
    EXTRACT(YEAR FROM TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd'))
                                                         AS launch_year
  FROM "built_enriched_weekly" w
  WHERE
    w.parent_brand = 'BUILT'
    AND w.military_excluded_flag = 0
    AND w.first_week_selling IS NOT NULL
)

SELECT
  launch_date                                            AS __time,
  upc, description, channel_outlet, retail_account,
  geography_raw, geography_display, geography_level,
  parent_brand, brand_line,
  specific_flavor_normalized, spins_flavor, pack_count,
  first_week_selling,

  -- ── 4-WEEK WINDOW ─────────────────────────────────────────────
  SUM(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN base_units ELSE 0 END)
                                                         AS pre_4w_base_units,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN base_units ELSE 0 END)
                                                         AS post_4w_base_units,
  SUM(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN tdp        ELSE 0 END)
                                                         AS pre_4w_tdp,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN tdp        ELSE 0 END)
                                                         AS post_4w_tdp,
  AVG(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN avg_weekly_units_spm END)
                                                         AS pre_4w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN avg_weekly_units_spm END)
                                                         AS post_4w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN avg_weekly_units_per_store_selling_per_item END)
                                                         AS pre_4w_velocity_store_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN avg_weekly_units_per_store_selling_per_item END)
                                                         AS post_4w_velocity_store_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN pct_stores_selling END)
                                                         AS pre_4w_pct_stores_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN pct_stores_selling END)
                                                         AS post_4w_pct_stores_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN arp END)
                                                         AS pre_4w_arp,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN arp END)
                                                         AS post_4w_arp,
  SUM(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN promo_weeks ELSE 0 END)
                                                         AS pre_4w_promo_weeks,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN promo_weeks ELSE 0 END)
                                                         AS post_4w_promo_weeks,
  AVG(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN units_pct_promo END)
                                                         AS pre_4w_units_pct_promo,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN units_pct_promo END)
                                                         AS post_4w_units_pct_promo,
  COUNT(CASE WHEN weeks_from_launch BETWEEN -4 AND -1 THEN 1 END)
                                                         AS pre_4w_weeks_count,
  COUNT(CASE WHEN weeks_from_launch BETWEEN  0 AND  3 THEN 1 END)
                                                         AS post_4w_weeks_count,

  -- ── 13-WEEK WINDOW ────────────────────────────────────────────
  SUM(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN base_units ELSE 0 END)
                                                         AS pre_13w_base_units,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN base_units ELSE 0 END)
                                                         AS post_13w_base_units,
  -- Sequential baseline: 13 weeks before the pre window
  SUM(CASE WHEN weeks_from_launch BETWEEN -26 AND -14 THEN base_units ELSE 0 END)
                                                         AS prior_13w_base_units,
  -- Year-ago: pull from Yago columns over the same post window dates
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN base_units_yago ELSE 0 END)
                                                         AS yago_13w_base_units,
  SUM(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN tdp ELSE 0 END)
                                                         AS pre_13w_tdp,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN tdp ELSE 0 END)
                                                         AS post_13w_tdp,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN avg_weekly_units_spm END)
                                                         AS pre_13w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN avg_weekly_units_spm END)
                                                         AS post_13w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN -26 AND -14 THEN avg_weekly_units_spm END)
                                                         AS prior_13w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN avg_weekly_units_per_store_selling_per_item END)
                                                         AS post_13w_velocity_store_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN avg_weekly_units_per_store_selling_per_item END)
                                                         AS pre_13w_velocity_store_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN pct_stores_selling END)
                                                         AS pre_13w_pct_stores_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN pct_stores_selling END)
                                                         AS post_13w_pct_stores_selling,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN arp END)
                                                         AS pre_13w_arp,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN arp END)
                                                         AS post_13w_arp,
  SUM(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN promo_weeks ELSE 0 END)
                                                         AS pre_13w_promo_weeks,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN promo_weeks ELSE 0 END)
                                                         AS post_13w_promo_weeks,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN units_pct_promo END)
                                                         AS pre_13w_units_pct_promo,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN units_pct_promo END)
                                                         AS post_13w_units_pct_promo,
  AVG(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN arp_pct_discount_any_promo END)
                                                         AS pre_13w_arp_discount,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN arp_pct_discount_any_promo END)
                                                         AS post_13w_arp_discount,
  COUNT(CASE WHEN weeks_from_launch BETWEEN -13 AND -1  THEN 1 END)
                                                         AS pre_13w_weeks_count,
  COUNT(CASE WHEN weeks_from_launch BETWEEN  0  AND 12  THEN 1 END)
                                                         AS post_13w_weeks_count,

  -- ── 26-WEEK WINDOW ────────────────────────────────────────────
  SUM(CASE WHEN weeks_from_launch BETWEEN -26 AND -1  THEN base_units ELSE 0 END)
                                                         AS pre_26w_base_units,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 25  THEN base_units ELSE 0 END)
                                                         AS post_26w_base_units,
  AVG(CASE WHEN weeks_from_launch BETWEEN -26 AND -1  THEN avg_weekly_units_spm END)
                                                         AS pre_26w_velocity_spm,
  AVG(CASE WHEN weeks_from_launch BETWEEN  0  AND 25  THEN avg_weekly_units_spm END)
                                                         AS post_26w_velocity_spm,
  SUM(CASE WHEN weeks_from_launch BETWEEN -26 AND -1  THEN tdp ELSE 0 END)
                                                         AS pre_26w_tdp,
  SUM(CASE WHEN weeks_from_launch BETWEEN  0  AND 25  THEN tdp ELSE 0 END)
                                                         AS post_26w_tdp,
  COUNT(CASE WHEN weeks_from_launch BETWEEN -26 AND -1  THEN 1 END)
                                                         AS pre_26w_weeks_count,
  COUNT(CASE WHEN weeks_from_launch BETWEEN  0  AND 25  THEN 1 END)
                                                         AS post_26w_weeks_count,

  -- ── YTD WINDOW (calendar year of launch) ──────────────────────
  SUM(CASE
    WHEN EXTRACT(YEAR FROM __time) = launch_year
     AND weeks_from_launch < 0
      THEN base_units ELSE 0 END)                        AS pre_ytd_base_units,
  SUM(CASE
    WHEN EXTRACT(YEAR FROM __time) = launch_year
     AND weeks_from_launch >= 0
      THEN base_units ELSE 0 END)                        AS post_ytd_base_units,
  COUNT(CASE
    WHEN EXTRACT(YEAR FROM __time) = launch_year
     AND weeks_from_launch < 0 THEN 1 END)               AS pre_ytd_weeks_count,
  COUNT(CASE
    WHEN EXTRACT(YEAR FROM __time) = launch_year
     AND weeks_from_launch >= 0 THEN 1 END)              AS post_ytd_weeks_count

FROM weekly_tagged
GROUP BY
  upc, description, channel_outlet, retail_account,
  geography_raw, geography_display, geography_level,
  parent_brand, brand_line,
  specific_flavor_normalized, spins_flavor, pack_count,
  first_week_selling, launch_date
PARTITIONED BY DAY
CLUSTERED BY upc, channel_outlet, retail_account, geography_raw
```

**Verify:** `SELECT COUNT(DISTINCT upc) FROM built_prepost_features` should match BUILT SKU count. Check `pre_13w_weeks_count` is non-zero for established SKUs.

## Q4 Donor pre/post aggregation

**Purpose:** Same window logic as Q3, applied to each donor candidate but keyed to the **focal SKU's** `first_week_selling`. Aligns the donor's pre/post window to the focal's launch event, not the donor's own history.

**Context required:** Set `{"sqlJoinAlgorithm": "sortMerge"}` in the query context panel — this query joins two large materialized tables and is likely to exceed the broadcast join memory limit.

```sql
SET sqlJoinAlgorithm      = 'sortMerge';           -- REQUIRED: broadcast join exceeds 311MB limit (BroadcastTablesTooLarge)
SET sqlSortMergeDiskBuffered = 'true';              -- spills sort-merge merge buffers to disk; reduces per-worker memory pressure
SET durableShuffleStorage = 'true';               -- routes shuffle files to S3; cluster-level config completed by Rob
SET maxNumTasks           = 4;                     -- REQUIRED: single-worker sort-merge stalls (E13); 15 workers confirmed available
```

## Q5 Final ML feature table assembly

**Purpose:** Join focal and donor pre/post features. Compute all derived change signals, percentage deltas, promo confound flag, deterministic labels, `incremental_share`, and `cannibalization_rate`. One row = one (focal_upc, donor_upc, geography, window_type) training example.

**`Units/TDP` is never computed anywhere in this query.**

```sql
SET sqlJoinAlgorithm      = 'sortMerge';           -- REQUIRED: broadcast join exceeds 311MB limit (BroadcastTablesTooLarge)
SET sqlSortMergeDiskBuffered = 'true';              -- spills sort-merge merge buffers to disk; reduces per-worker memory pressure
SET durableShuffleStorage = 'true';               -- routes shuffle files to S3; cluster-level config completed by Rob
SET maxNumTasks           = 4;                     -- REQUIRED: single-worker sort-merge stalls (E13); 15 workers confirmed available
-- SET rowsPerSegment        = 5000000;           -- output segment size tuning (minor; uncomment when Q2 succeeds)

REPLACE INTO "ml_training_features"
OVERWRITE ALL
WITH joined AS (
  SELECT
    f.upc                                                AS focal_upc,
    f.description                                        AS focal_description,
    f.channel_outlet, f.retail_account,
    f.geography_raw, f.geography_display, f.geography_level,
    f.parent_brand                                       AS focal_parent_brand,
    f.brand_line                                         AS focal_brand_line,
    f.specific_flavor_normalized                         AS focal_flavor,
    f.spins_flavor                                       AS focal_spins_flavor,
    f.pack_count                                         AS focal_pack_count,
    f.first_week_selling,
    '13w'                                                AS window_type,

    -- Focal pre/post metrics (13w default training window)
    f.pre_13w_base_units, f.post_13w_base_units,
    f.prior_13w_base_units, f.yago_13w_base_units,
    f.pre_13w_tdp, f.post_13w_tdp,
    f.pre_13w_velocity_spm, f.post_13w_velocity_spm,
    f.pre_13w_velocity_store_selling, f.post_13w_velocity_store_selling,
    f.pre_13w_pct_stores_selling, f.post_13w_pct_stores_selling,
    f.pre_13w_arp, f.post_13w_arp,
    f.pre_13w_promo_weeks, f.post_13w_promo_weeks,
    f.pre_13w_units_pct_promo, f.post_13w_units_pct_promo,
    f.pre_13w_arp_discount, f.post_13w_arp_discount,
    f.pre_13w_weeks_count                                AS focal_pre_weeks_count,
    f.post_13w_weeks_count                               AS focal_post_weeks_count,

    -- Donor metrics
    d.candidate_upc                                      AS donor_upc,
    d.candidate_description                              AS donor_description,
    d.candidate_flavor                                   AS donor_flavor,
    d.candidate_spins_flavor                             AS donor_spins_flavor,
    d.candidate_pack_count                               AS donor_pack_count,
    d.candidate_brand                                    AS donor_brand,
    d.candidate_brand_line                               AS donor_brand_line,
    d.relationship_distance, d.comparison_type, d.pack_distance,
    d.donor_pre_13w_base_units, d.donor_post_13w_base_units,
    d.donor_yago_13w_base_units,
    d.donor_pre_13w_velocity_spm, d.donor_post_13w_velocity_spm,
    d.donor_pre_13w_velocity_store_selling, d.donor_post_13w_velocity_store_selling,
    d.donor_pre_13w_pct_stores_selling, d.donor_post_13w_pct_stores_selling,
    d.donor_pre_13w_arp, d.donor_post_13w_arp,
    d.donor_pre_13w_promo_weeks, d.donor_post_13w_promo_weeks,
    d.donor_pre_13w_units_pct_promo, d.donor_post_13w_units_pct_promo,
    d.donor_pre_13w_arp_discount, d.donor_post_13w_arp_discount,
    d.donor_pre_13w_weeks_count, d.donor_post_13w_weeks_count

  FROM "built_prepost_features" f
  JOIN "donor_prepost_features" d
    ON  f.upc              = d.focal_upc
    AND f.channel_outlet   = d.channel_outlet
    AND f.retail_account   = d.retail_account
    AND f.geography_raw    = d.geography_raw
    AND f.first_week_selling = d.focal_first_week_selling
)

SELECT
  TIME_PARSE(first_week_selling, 'yyyy-MM-dd')           AS __time,
  focal_upc, focal_description,
  channel_outlet, retail_account,
  geography_raw, geography_display, geography_level,
  focal_parent_brand, focal_brand_line,
  focal_flavor, focal_spins_flavor, focal_pack_count,
  donor_upc, donor_description,
  donor_flavor, donor_spins_flavor, donor_pack_count,
  donor_brand, donor_brand_line,
  relationship_distance, comparison_type, pack_distance,
  window_type, first_week_selling,
  focal_pre_weeks_count, focal_post_weeks_count,
  donor_pre_13w_weeks_count, donor_post_13w_weeks_count,

  -- ── DERIVED CHANGE SIGNALS ────────────────────────────────────
  (post_13w_base_units - pre_13w_base_units)
    / NULLIF(pre_13w_base_units, 0)                      AS focal_base_units_pct_chg,

  (donor_post_13w_base_units - donor_pre_13w_base_units)
    / NULLIF(donor_pre_13w_base_units, 0)                AS donor_base_units_pct_chg,

  -- Net transfer: focal gain + donor loss (negative donor = net positive transfer)
  (post_13w_base_units - pre_13w_base_units)
  + (donor_post_13w_base_units - donor_pre_13w_base_units)
                                                         AS base_units_delta_diff,

  (post_13w_tdp - pre_13w_tdp)
    / NULLIF(pre_13w_tdp, 0)                             AS focal_tdp_pct_chg,

  (post_13w_velocity_spm - pre_13w_velocity_spm)
    / NULLIF(pre_13w_velocity_spm, 0)                    AS focal_velocity_spm_pct_chg,

  (donor_post_13w_velocity_spm - donor_pre_13w_velocity_spm)
    / NULLIF(donor_pre_13w_velocity_spm, 0)              AS donor_velocity_spm_pct_chg,

  (post_13w_velocity_spm - pre_13w_velocity_spm)
  + (donor_post_13w_velocity_spm - donor_pre_13w_velocity_spm)
                                                         AS velocity_spm_delta_diff,

  (post_13w_velocity_store_selling - pre_13w_velocity_store_selling)
    / NULLIF(pre_13w_velocity_store_selling, 0)          AS focal_velocity_store_selling_pct_chg,

  (donor_post_13w_velocity_store_selling - donor_pre_13w_velocity_store_selling)
    / NULLIF(donor_pre_13w_velocity_store_selling, 0)    AS donor_velocity_store_selling_pct_chg,

  (post_13w_pct_stores_selling - pre_13w_pct_stores_selling)
    / NULLIF(pre_13w_pct_stores_selling, 0)              AS focal_pct_stores_selling_pct_chg,

  (donor_post_13w_pct_stores_selling - donor_pre_13w_pct_stores_selling)
    / NULLIF(donor_pre_13w_pct_stores_selling, 0)        AS donor_pct_stores_selling_pct_chg,

  (post_13w_arp - pre_13w_arp)
    / NULLIF(pre_13w_arp, 0)                             AS focal_arp_pct_chg,

  post_13w_promo_weeks - pre_13w_promo_weeks             AS focal_promo_week_delta,
  donor_post_13w_promo_weeks - donor_pre_13w_promo_weeks AS donor_promo_week_delta,

  -- Promo confounded flag
  CASE
    WHEN ABS(post_13w_promo_weeks - pre_13w_promo_weeks) > 2
      OR ABS(post_13w_arp_discount - pre_13w_arp_discount) > 0.05
      OR ABS(donor_post_13w_promo_weeks - donor_pre_13w_promo_weeks) > 2
      THEN 1
    ELSE 0
  END AS promo_confounded,

  -- Price per unit (controls for pack format value differences)
  post_13w_arp / NULLIF(focal_pack_count, 0)             AS focal_price_per_unit,
  donor_post_13w_arp / NULLIF(donor_pack_count, 0)       AS donor_price_per_unit,

  -- Year-ago seasonality controls
  (post_13w_base_units - yago_13w_base_units)
    / NULLIF(yago_13w_base_units, 0)                     AS focal_base_units_yago_pct_chg,

  (donor_post_13w_base_units - donor_yago_13w_base_units)
    / NULLIF(donor_yago_13w_base_units, 0)               AS donor_base_units_yago_pct_chg,

  -- Raw metric pass-throughs for Python scoring
  post_13w_base_units                                    AS focal_post_base_units,
  pre_13w_base_units                                     AS focal_pre_base_units,
  post_13w_tdp                                           AS focal_post_tdp,
  pre_13w_tdp                                            AS focal_pre_tdp,
  post_13w_velocity_spm                                  AS focal_post_velocity_spm,
  pre_13w_velocity_spm                                   AS focal_pre_velocity_spm,
  post_13w_promo_weeks                                   AS focal_post_promo_weeks,
  pre_13w_promo_weeks                                    AS focal_pre_promo_weeks,
  post_13w_units_pct_promo                               AS focal_post_units_pct_promo,
  pre_13w_units_pct_promo                                AS focal_pre_units_pct_promo,
  post_13w_arp_discount                                  AS focal_post_arp_discount,
  pre_13w_arp_discount                                   AS focal_pre_arp_discount,
  donor_post_13w_base_units, donor_pre_13w_base_units,
  donor_post_13w_velocity_spm, donor_pre_13w_velocity_spm,
  donor_post_13w_promo_weeks, donor_pre_13w_promo_weeks,
  donor_post_13w_units_pct_promo, donor_pre_13w_units_pct_promo,
  donor_post_13w_arp_discount, donor_pre_13w_arp_discount,

  -- ── DETERMINISTIC LABELS ─────────────────────────────────────
  CASE
    WHEN (donor_post_13w_base_units - donor_pre_13w_base_units)
           / NULLIF(donor_pre_13w_base_units, 0) < -0.10
     AND  (post_13w_base_units - pre_13w_base_units)
           / NULLIF(pre_13w_base_units, 0) > 0.03
      THEN 'CANNIBALIZING'
    WHEN (donor_post_13w_base_units - donor_pre_13w_base_units)
           / NULLIF(donor_pre_13w_base_units, 0) < -0.03
     AND  (post_13w_base_units - pre_13w_base_units)
           / NULLIF(pre_13w_base_units, 0) > 0
      THEN 'WATCH'
    WHEN (donor_post_13w_base_units - donor_pre_13w_base_units)
           / NULLIF(donor_pre_13w_base_units, 0) >= -0.03
      THEN 'INCREMENTAL'
    ELSE 'NEUTRAL'
  END AS label_deterministic,

  CASE
    WHEN (post_13w_velocity_spm - pre_13w_velocity_spm)
           / NULLIF(pre_13w_velocity_spm, 0) > 0
     AND  (post_13w_base_units - pre_13w_base_units)
           / NULLIF(pre_13w_base_units, 0) > 0
      THEN 'DEMAND_LED'
    WHEN (post_13w_tdp - pre_13w_tdp)
           / NULLIF(pre_13w_tdp, 0) > 0.15
     AND  (post_13w_velocity_spm - pre_13w_velocity_spm)
           / NULLIF(pre_13w_velocity_spm, 0) <= 0
      THEN 'DISTRIBUTION_LED'
    ELSE 'MIXED'
  END AS demand_vs_dist,

  -- Incremental share: focal gain / (focal gain + ABS donor loss)
  CASE
    WHEN (post_13w_base_units - pre_13w_base_units) > 0
      THEN GREATEST(0.0, LEAST(1.0,
        (post_13w_base_units - pre_13w_base_units) /
        NULLIF(
          (post_13w_base_units - pre_13w_base_units)
          + ABS(LEAST(0.0, donor_post_13w_base_units - donor_pre_13w_base_units)),
          0)
      ))
    ELSE NULL
  END AS incremental_share,

  -- Cannibalization rate: donor units lost / focal units gained, bounded 0-1
  CASE
    WHEN (post_13w_base_units - pre_13w_base_units) > 0
      THEN GREATEST(0.0, LEAST(1.0,
        ABS(LEAST(0.0, donor_post_13w_base_units - donor_pre_13w_base_units))
        / NULLIF(post_13w_base_units - pre_13w_base_units, 0)
      ))
    ELSE NULL
  END AS cannibalization_rate

FROM joined
WHERE
  focal_pre_weeks_count  >= 8
  AND focal_post_weeks_count >= 8
  AND donor_pre_13w_weeks_count >= 8
  AND donor_post_13w_weeks_count >= 8
  AND pre_13w_base_units > 0
  AND post_13w_tdp > 0
  AND donor_pre_13w_base_units > 0
PARTITIONED BY DAY
CLUSTERED BY focal_upc, donor_upc, channel_outlet, retail_account, geography_raw
```

**Verify:** `SELECT label_deterministic, COUNT(*) FROM ml_training_features GROUP BY 1` — INCREMENTAL should dominate; some WATCH and CANNIBALIZING; NEUTRAL excluded.

## Q6 Rolling stats + z-scores

**Purpose:** Compute rolling 4w, 8w, and 13w averages, standard deviations, and z-scores for `base_units`, `avg_weekly_units_spm`, and `tdp`. Powers Layer 1 of the two-layer event detector. Window functions require Druid 0.22+ with SQL windowing enabled.

```sql
REPLACE INTO "event_detection_weekly"
OVERWRITE ALL
WITH base AS (
  SELECT
    __time,
    upc, description,
    channel_outlet, retail_account,
    geography_raw, geography_display, geography_level,
    parent_brand, specific_flavor_normalized, spins_flavor, pack_count,
    base_units, avg_weekly_units_spm, tdp,
    pct_stores_selling, stores_selling,

    -- Rolling 4w base_units
    AVG(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
    ) AS base_units_roll4_avg,
    STDDEV(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
    ) AS base_units_roll4_std,

    -- Rolling 8w base_units (primary z-score baseline)
    AVG(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS base_units_roll8_avg,
    STDDEV(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS base_units_roll8_std,

    -- Rolling 13w base_units
    AVG(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 12 PRECEDING AND CURRENT ROW
    ) AS base_units_roll13_avg,
    STDDEV(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 12 PRECEDING AND CURRENT ROW
    ) AS base_units_roll13_std,

    -- Rolling 8w velocity (avg_weekly_units_spm)
    AVG(avg_weekly_units_spm) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS velocity_spm_roll8_avg,
    STDDEV(avg_weekly_units_spm) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS velocity_spm_roll8_std,

    -- Rolling 8w TDP
    AVG(tdp) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS tdp_roll8_avg,
    STDDEV(tdp) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
      ROWS BETWEEN 7 PRECEDING AND CURRENT ROW
    ) AS tdp_roll8_std,

    -- Week-over-week deltas
    base_units - LAG(base_units) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
    ) AS base_units_wow_delta,
    avg_weekly_units_spm - LAG(avg_weekly_units_spm) OVER (
      PARTITION BY upc, channel_outlet, retail_account, geography_raw
      ORDER BY __time
    ) AS velocity_spm_wow_delta

  FROM "built_enriched_weekly"
  WHERE parent_brand = 'BUILT' AND military_excluded_flag = 0
)

SELECT
  __time,
  upc, description,
  channel_outlet, retail_account,
  geography_raw, geography_display, geography_level,
  parent_brand, specific_flavor_normalized, spins_flavor, pack_count,
  base_units, avg_weekly_units_spm, tdp,
  pct_stores_selling, stores_selling,

  -- Rolling stats (passed to Python Welch t-test)
  base_units_roll4_avg, base_units_roll4_std,
  base_units_roll8_avg, base_units_roll8_std,
  base_units_roll13_avg, base_units_roll13_std,
  velocity_spm_roll8_avg, velocity_spm_roll8_std,
  tdp_roll8_avg, tdp_roll8_std,
  base_units_wow_delta, velocity_spm_wow_delta,

  -- Z-scores vs 8-week rolling baseline
  CASE
    WHEN base_units_roll8_std > 0
      THEN (base_units - base_units_roll8_avg) / base_units_roll8_std
    ELSE 0
  END AS base_units_z8,

  CASE
    WHEN velocity_spm_roll8_std > 0
      THEN (avg_weekly_units_spm - velocity_spm_roll8_avg) / velocity_spm_roll8_std
    ELSE 0
  END AS velocity_spm_z8,

  CASE
    WHEN tdp_roll8_std > 0
      THEN (tdp - tdp_roll8_avg) / tdp_roll8_std
    ELSE 0
  END AS tdp_z8,

  -- Outlier classification (Layer 1 gate — both base_units and velocity)
  CASE
    WHEN ABS(CASE WHEN base_units_roll8_std > 0
      THEN (base_units - base_units_roll8_avg) / base_units_roll8_std
      ELSE 0 END) >= 3.0 THEN 'EXTREME_OUTLIER'
    WHEN ABS(CASE WHEN base_units_roll8_std > 0
      THEN (base_units - base_units_roll8_avg) / base_units_roll8_std
      ELSE 0 END) >= 2.0 THEN 'OUTLIER'
    WHEN ABS(CASE WHEN base_units_roll8_std > 0
      THEN (base_units - base_units_roll8_avg) / base_units_roll8_std
      ELSE 0 END) >= 1.5 THEN 'WATCH'
    ELSE 'NORMAL'
  END AS base_units_outlier_class,

  CASE
    WHEN ABS(CASE WHEN velocity_spm_roll8_std > 0
      THEN (avg_weekly_units_spm - velocity_spm_roll8_avg) / velocity_spm_roll8_std
      ELSE 0 END) >= 3.0 THEN 'EXTREME_OUTLIER'
    WHEN ABS(CASE WHEN velocity_spm_roll8_std > 0
      THEN (avg_weekly_units_spm - velocity_spm_roll8_avg) / velocity_spm_roll8_std
      ELSE 0 END) >= 2.0 THEN 'OUTLIER'
    WHEN ABS(CASE WHEN velocity_spm_roll8_std > 0
      THEN (avg_weekly_units_spm - velocity_spm_roll8_avg) / velocity_spm_roll8_std
      ELSE 0 END) >= 1.5 THEN 'WATCH'
    ELSE 'NORMAL'
  END AS velocity_outlier_class

FROM base
PARTITIONED BY DAY
CLUSTERED BY upc, channel_outlet, retail_account, geography_raw
```

**Verify:** `SELECT upc, MAX(ABS(base_units_z8)) AS max_z FROM event_detection_weekly GROUP BY upc ORDER BY max_z DESC LIMIT 10`. Rows with |z| > 2.0 are Layer 1 event candidates.

## Q8 New UPC classification

**Purpose:** Classify each new UPC as `NEW_PACK_SIZE`, `NEW_FLAVOR_CANDIDATE`, or `DUPLICATE_OR_RELAUNCH`. Only `NEW_PACK_SIZE` rows with no flavor taxonomy conflict auto-enroll in pack ladder scoring.

```sql
REPLACE INTO "new_upc_classifications"
OVERWRITE ALL
WITH existing_catalog AS (
  SELECT DISTINCT
    upc, specific_flavor_normalized,
    spins_flavor_canonical, spins_flavor,
    pack_count, description
  FROM "built_enriched_weekly"
  WHERE parent_brand = 'BUILT' AND military_excluded_flag = 0
    AND upc NOT IN (SELECT upc FROM "new_upc_candidates")
)

SELECT
  n.upc, n.description, n.source_brand,
  n.spins_flavor_raw, n.spins_flavor_mapped,
  n.spins_flavor_canonical, n.spins_flavor,
  n.specific_flavor_normalized,
  n.pack_count, n.size_oz,
  n.first_seen_week, n.manual_review_needed, n.detected_at,

  -- Classification
  CASE
    WHEN EXISTS (
      SELECT 1 FROM existing_catalog e
      WHERE e.specific_flavor_normalized = n.specific_flavor_normalized
        AND e.spins_flavor_canonical     = n.spins_flavor_canonical
        AND e.pack_count                != n.pack_count
        AND n.spins_flavor_mapped        = n.spins_flavor_canonical
    )
    AND n.pack_count IN (1,2,3,4,5,6,8,10,12,15,16,18,20,24)
      THEN 'NEW_PACK_SIZE'
    WHEN EXISTS (
      SELECT 1 FROM existing_catalog e
      WHERE e.specific_flavor_normalized = n.specific_flavor_normalized
        AND e.pack_count                 = n.pack_count
    )
      THEN 'DUPLICATE_OR_RELAUNCH'
    ELSE 'NEW_FLAVOR_CANDIDATE'
  END AS upc_classification,

  -- Closest pack-ladder partner (populated for NEW_PACK_SIZE rows only)
  (
    SELECT e.upc FROM existing_catalog e
    WHERE e.specific_flavor_normalized = n.specific_flavor_normalized
      AND e.spins_flavor_canonical     = n.spins_flavor_canonical
      AND e.pack_count                != n.pack_count
    ORDER BY ABS(e.pack_count - n.pack_count)
    LIMIT 1
  ) AS closest_pack_partner_upc,

  (
    SELECT e.description FROM existing_catalog e
    WHERE e.specific_flavor_normalized = n.specific_flavor_normalized
      AND e.spins_flavor_canonical     = n.spins_flavor_canonical
      AND e.pack_count                != n.pack_count
    ORDER BY ABS(e.pack_count - n.pack_count)
    LIMIT 1
  ) AS closest_pack_partner_description,

  -- Taxonomy conflict: mapped FLAVOR != canonical FLAVOR
  CASE
    WHEN n.spins_flavor_mapped != n.spins_flavor_canonical THEN 'Y'
    ELSE 'N'
  END AS flavor_taxonomy_conflict

FROM "new_upc_candidates" n
PARTITIONED BY DAY
CLUSTERED BY upc
```

**Verify:** Any `NEW_PACK_SIZE` row with `flavor_taxonomy_conflict = 'Y'` must be reviewed before pack-ladder enrollment. These rows go to `taxonomy_review_queue` and are not scored until canonical FLAVOR placement is approved.

## Q9 Ramp monitoring

**Purpose:** For all BUILT SKUs in their first 16 weeks post-launch, compute `weeks_since_launch`, `scoring_status`, `distribution_trend`, `underperforming_flag`, and the UI ramp ribbon text. This is the single source of truth for ramp state — the Python scoring job reads `scoring_status` directly from this table.

```sql
REPLACE INTO "new_product_ramp_monitor"
OVERWRITE ALL
WITH ramp_base AS (
  SELECT
    w.__time,
    w.upc, w.description,
    w.channel_outlet, w.retail_account,
    w.geography_raw, w.geography_display, w.geography_level,
    w.parent_brand, w.brand_line,
    w.specific_flavor_normalized, w.spins_flavor, w.pack_count,
    w.first_week_selling,
    TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd')       AS launch_date,
    TIMESTAMPDIFF(WEEK,
      TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd'),
      w.__time)                                          AS weeks_since_launch,
    w.tdp, w.avg_weekly_units_spm,
    w.avg_weekly_units_per_store_selling_per_item,
    w.base_units, w.units, w.pct_stores_selling, w.stores_selling, w.arp,

    -- Peak TDP so far (running max)
    MAX(w.tdp) OVER (
      PARTITION BY w.upc, w.channel_outlet, w.retail_account, w.geography_raw
      ORDER BY w.__time
      ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS peak_tdp,

    -- Rolling 4-week average velocity
    AVG(w.avg_weekly_units_spm) OVER (
      PARTITION BY w.upc, w.channel_outlet, w.retail_account, w.geography_raw
      ORDER BY w.__time
      ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
    ) AS rolling_4w_avg_velocity_spm,

    -- TDP trend: 4-week change (positive = RAMPING, zero = STABLE, negative = DECLINING)
    w.tdp - LAG(w.tdp, 4) OVER (
      PARTITION BY w.upc, w.channel_outlet, w.retail_account, w.geography_raw
      ORDER BY w.__time
    ) AS tdp_4w_change,

    -- Velocity ratio vs rolling average (for underperforming detection)
    w.avg_weekly_units_spm / NULLIF(
      AVG(w.avg_weekly_units_spm) OVER (
        PARTITION BY w.upc, w.channel_outlet, w.retail_account, w.geography_raw
        ORDER BY w.__time
        ROWS BETWEEN 3 PRECEDING AND CURRENT ROW
      ), 0)                                              AS velocity_vs_4w_avg_ratio

  FROM "built_enriched_weekly" w
  WHERE
    w.parent_brand = 'BUILT'
    AND w.military_excluded_flag = 0
    AND w.first_week_selling IS NOT NULL
    AND TIMESTAMPDIFF(WEEK,
          TIME_PARSE(w.first_week_selling, 'yyyy-MM-dd'),
          w.__time) BETWEEN 0 AND 15
)

SELECT
  __time,
  upc, description,
  channel_outlet, retail_account,
  geography_raw, geography_display, geography_level,
  parent_brand, brand_line,
  specific_flavor_normalized, spins_flavor, pack_count,
  first_week_selling, launch_date,
  weeks_since_launch,
  tdp, avg_weekly_units_spm,
  avg_weekly_units_per_store_selling_per_item,
  base_units, pct_stores_selling, stores_selling, arp,
  peak_tdp, rolling_4w_avg_velocity_spm,
  tdp_4w_change, velocity_vs_4w_avg_ratio,

  -- Distribution trend
  CASE
    WHEN tdp_4w_change IS NULL OR weeks_since_launch < 4 THEN 'RAMPING'
    WHEN tdp_4w_change > 0                              THEN 'RAMPING'
    WHEN tdp_4w_change = 0                              THEN 'STABLE'
    WHEN tdp_4w_change < 0                              THEN 'DECLINING'
    ELSE 'RAMPING'
  END AS distribution_trend,

  -- Scoring status — single source of truth; Python reads this column directly
  CASE
    WHEN weeks_since_launch BETWEEN 0 AND 5  THEN 'SUPPRESSED'
    WHEN weeks_since_launch BETWEEN 6 AND 7  THEN 'LOW_CONFIDENCE'
    ELSE 'ACTIVE'
  END AS scoring_status,

  -- Ramp confidence level for scored rows
  CASE
    WHEN weeks_since_launch BETWEEN 0 AND 5  THEN NULL
    WHEN weeks_since_launch BETWEEN 6 AND 7  THEN 'Low'
    WHEN weeks_since_launch BETWEEN 8 AND 11 THEN 'Medium'
    ELSE 'Medium'
  END AS ramp_confidence,

  -- UI ramp ribbon text (displayed on Ramp Monitor screen)
  CASE
    WHEN weeks_since_launch BETWEEN 0 AND 5
      THEN 'RAMP WEEK ' || CAST(weeks_since_launch + 1 AS VARCHAR)
           || ' of 13 · Cannibalization scoring suppressed'
    WHEN weeks_since_launch BETWEEN 6 AND 7
      THEN 'RAMP WEEK ' || CAST(weeks_since_launch + 1 AS VARCHAR)
           || ' of 13 · Early signal · Low confidence'
    WHEN weeks_since_launch BETWEEN 8 AND 11
      THEN 'RAMP WEEK ' || CAST(weeks_since_launch + 1 AS VARCHAR)
           || ' of 13 · Signal maturing'
    ELSE 'RAMP WEEK ' || CAST(weeks_since_launch + 1 AS VARCHAR)
           || ' of 13 · Scoring fully active'
  END AS ui_ramp_ribbon,

  -- Underperforming flag: fires "Launch underperforming" event
  -- Requires: past suppression window (week 6+), velocity weak, TDP not ramping
  CASE
    WHEN weeks_since_launch >= 6
     AND velocity_vs_4w_avg_ratio < 0.85
     AND (tdp_4w_change IS NOT NULL AND tdp_4w_change <= 0)
      THEN 1
    WHEN weeks_since_launch >= 8
     AND velocity_vs_4w_avg_ratio < 0.85
      THEN 1
    ELSE 0
  END AS underperforming_flag,

  -- TDP ratio current/peak for Ramp Monitor "TDP now / peak" column
  CAST(tdp AS DOUBLE) / NULLIF(CAST(peak_tdp AS DOUBLE), 0)
                                                         AS tdp_ratio_to_peak,

  CAST(CAST(tdp AS BIGINT) AS VARCHAR)
  || ' / '
  || CAST(CAST(peak_tdp AS BIGINT) AS VARCHAR)           AS tdp_now_slash_peak

FROM ramp_base
PARTITIONED BY DAY
```

**Verify:** `SELECT scoring_status, COUNT(*) FROM new_product_ramp_monitor GROUP BY 1`. All rows with `weeks_since_launch` 0–5 must show `SUPPRESSED`. Run `SELECT upc, MAX(weeks_since_launch) FROM new_product_ramp_monitor GROUP BY upc` to confirm the full 16-week window is populated.

## Q14 Price elasticity weekly features

**Purpose:** Add `price_per_bar`, log-transformed price and volume, `promo_depth_bucket`, and `promo_confounded` flag to each BUILT weekly row. Foundation for all downstream elasticity regression.

```sql
REPLACE INTO "price_elasticity_weekly_features"
OVERWRITE ALL
SELECT
  w.__time,
  w.upc, w.description,
  w.channel_outlet, w.retail_account,
  w.geography_raw, w.geography_display, w.geography_level,
  w.parent_brand, w.brand_line,
  w.specific_flavor_normalized, w.spins_flavor, w.pack_count,
  w.first_week_selling,

  -- Price per bar (primary elasticity unit)
  w.arp / NULLIF(w.pack_count, 0)                        AS price_per_bar,
  w.base_arp / NULLIF(w.pack_count, 0)                   AS base_price_per_bar,

  -- Log-transformed price and volume for log-log OLS regression
  LN(GREATEST(w.arp / NULLIF(w.pack_count, 0), 0.01))   AS ln_price_per_bar,
  LN(GREATEST(w.base_units, 1))                          AS ln_base_units,
  LN(GREATEST(w.avg_weekly_units_spm, 0.01))             AS ln_velocity_spm,

  -- Promo depth bucket
  CASE
    WHEN w.arp_pct_discount_any_promo >= 0.30 THEN '30pct_plus'
    WHEN w.arp_pct_discount_any_promo >= 0.20 THEN '20_30pct'
    WHEN w.arp_pct_discount_any_promo >= 0.10 THEN '10_20pct'
    WHEN w.arp_pct_discount_any_promo >  0    THEN '0_10pct'
    ELSE 'no_promo'
  END AS promo_depth_bucket,

  -- Promo confounded: exclude from own-price training; include in promo training
  CASE
    WHEN w.promo_weeks > 2 OR w.arp_pct_discount_any_promo > 0.05 THEN 1
    ELSE 0
  END AS promo_confounded,

  -- Seasonality controls
  EXTRACT(WEEK FROM w.__time)                            AS week_of_year,
  w.base_units_yago,
  (w.base_units - w.base_units_yago)
    / NULLIF(w.base_units_yago, 0)                       AS base_units_yago_pct_chg,

  -- Metric pass-throughs
  w.base_units, w.units,
  w.tdp, w.avg_weekly_tdp,
  w.avg_weekly_units_spm,
  w.avg_weekly_units_per_store_selling_per_item,
  w.pct_stores_selling, w.stores_selling,
  w.arp, w.base_arp, w.arp_yago,
  w.arp_pct_discount_any_promo,
  w.promo_weeks, w.units_pct_promo,
  w.units_lift_tpr, w.units_lift_any_display, w.units_lift_any_feature,
  w.tdp_any_promo, w.tdp_non_promo, w.incr_units

FROM "built_enriched_weekly" w
WHERE
  w.parent_brand = 'BUILT'
  AND w.military_excluded_flag = 0
  AND w.pack_count > 0
  AND w.arp > 0
PARTITIONED BY DAY
```

**Verify:** `SELECT MIN(price_per_bar), MAX(price_per_bar), AVG(price_per_bar) FROM price_elasticity_weekly_features`. BUILT 1ct ~$2.10–$2.60; 4pk ~$2.00–$2.20/bar; 12pk ~$1.70–$2.00/bar. Any value above $5.00/bar indicates a pack_count error — stop and fix before proceeding.

## Q15 Pack price ladder weekly

**Purpose:** Self-join to compare price-per-bar across all pack sizes of the same specific flavor. Produces one row per (focal_upc, partner_upc, week, geography) for all distance-1 pack pairs.

```sql
REPLACE INTO "price_pack_ladder_weekly"
OVERWRITE ALL
SELECT
  f.__time,
  f.channel_outlet, f.retail_account,
  f.geography_raw, f.geography_display, f.geography_level,

  -- Focal item
  f.upc                                                  AS focal_upc,
  f.description                                          AS focal_description,
  f.specific_flavor_normalized                           AS specific_flavor,
  f.spins_flavor,
  f.pack_count                                           AS focal_pack_count,
  f.price_per_bar                                        AS focal_price_per_bar,
  f.base_price_per_bar                                   AS focal_base_price_per_bar,
  f.arp                                                  AS focal_arp,
  f.base_units                                           AS focal_base_units,
  f.avg_weekly_units_spm                                 AS focal_velocity_spm,
  f.pct_stores_selling                                   AS focal_pct_stores_selling,

  -- Partner item (same specific flavor, different pack count)
  p.upc                                                  AS partner_upc,
  p.description                                          AS partner_description,
  p.pack_count                                           AS partner_pack_count,
  p.price_per_bar                                        AS partner_price_per_bar,
  p.arp                                                  AS partner_arp,
  p.base_units                                           AS partner_base_units,
  p.avg_weekly_units_spm                                 AS partner_velocity_spm,

  -- Per-bar price gap: positive = focal costs more per bar than partner
  (f.price_per_bar - p.price_per_bar)
    / NULLIF(p.price_per_bar, 0)                         AS price_per_bar_gap_pct,

  ABS(f.pack_count - p.pack_count)                       AS pack_distance,

  CASE
    WHEN f.pack_count < p.pack_count THEN 'TRADE_UP_AVAILABLE'
    WHEN f.pack_count > p.pack_count THEN 'TRADE_DOWN_AVAILABLE'
    ELSE 'SAME_PACK'
  END AS pack_direction,

  -- Trade-up savings: how much cheaper per bar is the larger pack?
  CASE
    WHEN p.pack_count > f.pack_count
      THEN (f.price_per_bar - p.price_per_bar) / NULLIF(f.price_per_bar, 0)
    ELSE NULL
  END AS trade_up_savings_pct,

  -- Ladder compression flag: < 10% per-bar gap = PACK_LADDER_COMPRESSION event
  CASE
    WHEN p.pack_count > f.pack_count
     AND (f.price_per_bar - p.price_per_bar)
           / NULLIF(f.price_per_bar, 0) < 0.10
      THEN 1
    ELSE 0
  END AS ladder_compression_flag

FROM "price_elasticity_weekly_features" f
JOIN "price_elasticity_weekly_features" p
  ON  f.__time                     = p.__time
  AND f.channel_outlet             = p.channel_outlet
  AND f.retail_account             = p.retail_account
  AND f.geography_raw              = p.geography_raw
  AND f.specific_flavor_normalized = p.specific_flavor_normalized
  AND f.spins_flavor               = p.spins_flavor
  AND f.parent_brand               = p.parent_brand
  AND f.upc                       != p.upc
PARTITIONED BY DAY
```

**Verify:** `SELECT focal_description, partner_description, AVG(price_per_bar_gap_pct) FROM price_pack_ladder_weekly GROUP BY 1,2 ORDER BY 1,2`. Mint Chip 4pk vs 1ct should show roughly −15% (4pk cheaper per bar).

```sql
  c.upc                                                  AS competitor_upc,
  c.description                                          AS competitor_description,
  c.source_brand                                         AS competitor_brand,
  CASE c.source_brand
    WHEN 'QUEST'                     THEN 1
    WHEN 'BAREBELLS'                 THEN 1
    WHEN 'ONE BAR'                   THEN 1
    WHEN 'RXBAR'                     THEN 1
    WHEN 'PERFECT BAR'               THEN 1
    WHEN 'THINK!'                    THEN 1
    WHEN 'PURE PROTEIN'              THEN 1
    WHEN 'POWER CRUNCH'              THEN 1
    WHEN 'NO COW'                    THEN 1
    WHEN 'ROBERT IRVINES FITCRUNCH'  THEN 1
    WHEN 'CLIF BUILDERS'             THEN 1
    WHEN 'KIND'                      THEN 2
    WHEN 'ATKINS'                    THEN 2
    WHEN 'LUNA'                      THEN 2
    WHEN 'GOMACRO'                   THEN 2
    WHEN 'ZONE PERFECT'              THEN 2
    WHEN 'MET-RX'                    THEN 2
    WHEN 'NUGO NUTRITION'            THEN 2
    WHEN 'ALOHA'                     THEN 2
    WHEN 'LENNY & LARRYS'            THEN 2
    WHEN 'PRIVATE LABEL'             THEN 2
    WHEN 'CLIF BAR'                  THEN 3
    WHEN 'LARABAR'                   THEN 3
    WHEN 'BOBOS'                     THEN 3
    WHEN 'CLIF ZBAR'                 THEN 3
    WHEN 'HONEY STINGER'             THEN 3
    WHEN 'CLIO'                      THEN 3
    WHEN 'THATS IT'                  THEN 3
    WHEN 'SPECIAL K'                 THEN 3
    WHEN 'GATORADE'                  THEN 3
    ELSE NULL
  END                                                    AS competitor_tier,
  c.spins_flavor_raw                                     AS competitor_spins_flavor_raw,
  c.source_pack_count                                    AS competitor_pack_count,
  c.arp / NULLIF(c.source_pack_count, 0)                AS competitor_price_per_bar,
  c.arp                                                  AS competitor_arp,
  c.base_units                                           AS competitor_base_units,
  c.avg_weekly_units_spm                                 AS competitor_velocity_spm,

  -- Price gap: positive = BUILT costs more per bar
  (b.price_per_bar - c.arp / NULLIF(c.source_pack_count, 0))
    / NULLIF(c.arp / NULLIF(c.source_pack_count, 0), 0) AS price_gap_pct_vs_competitor,

  -- Watch flag: gap >= 9% on Tier 1 same-FLAVOR competitor
  CASE
    WHEN ic.competitor_tier = 'TIER_1_DIRECT'
     AND b.spins_flavor_raw = c.spins_flavor_raw
     AND (b.price_per_bar - c.arp / NULLIF(c.source_pack_count, 0))
           / NULLIF(c.arp / NULLIF(c.source_pack_count, 0), 0) >= 0.09
      THEN 1
    ELSE 0
  END AS competitive_gap_watch_flag,

  CASE
    WHEN b.spins_flavor_raw = c.spins_flavor_raw        THEN 'SAME_SPINS_FLAVOR'
    WHEN b.spins_flavor     = c.spins_flavor_raw         THEN 'SAME_CANONICAL_FLAVOR'
    ELSE 'DIFFERENT_FLAVOR'
  END AS competitive_flavor_relationship

FROM "price_elasticity_weekly_features" b
JOIN "built_filtered_weekly" c
  ON  b.__time          = c.__time
  AND b.channel_outlet  = c.channel_outlet
  AND b.retail_account  = c.retail_account
  AND b.geography_raw   = c.geography_raw
  AND c.source_brand NOT IN ('BUILT','BUILT BAR','BUILT PUFF','BUILT SOUR PUFF')
  AND c.arp > 0
  AND c.source_pack_count > 0
-- item_catalog tier inlined as CASE to avoid sort-merge on a 30-row table
WHERE
  c.channel_outlet != 'CONVENTIONAL|MILITARY'
  AND c.retail_account != 'DECA'
PARTITIONED BY DAY
```

**Verify:** `SELECT competitor_brand, COUNT(*) FROM price_competitive_weekly GROUP BY 1 ORDER BY 2 DESC`. THINK!, RXBAR, QUEST, PERFECT BAR, NUGO NUTRITION should be present. Missing Tier 1 brands = absent from extract or brand name mismatch in `item_catalog`.

## Q17 Price elasticity training features

**Purpose:** Build regression-ready 13-week pre/post windows for own-price, cross-price, and promo elasticity estimation. One row per (focal_upc, window_end_date, geography). Python LightGBM reads this table to train the elasticity model.

```sql
REPLACE INTO "price_elasticity_training_features"
OVERWRITE ALL
WITH windows AS (
  SELECT
    w.__time                                             AS window_end_date,
    w.upc, w.description,
    w.channel_outlet, w.retail_account,
    w.geography_raw, w.geography_level,
    w.parent_brand, w.specific_flavor_normalized,
    w.spins_flavor, w.pack_count,

    -- Post 13w: 13 weeks ending at the current row's date
    AVG(p.price_per_bar) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_avg_price_per_bar,
    STDDEV(p.price_per_bar) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_std_price_per_bar,
    SUM(p.base_units) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_base_units,
    AVG(p.avg_weekly_units_spm) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_velocity_spm,
    SUM(p.promo_weeks) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_promo_weeks,
    AVG(p.arp_pct_discount_any_promo) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_arp_discount,
    AVG(p.tdp) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_tdp,
    COUNT(*) FILTER (
      WHERE p.__time BETWEEN TIMESTAMPADD(WEEK,-12,w.__time) AND w.__time
    ) AS post_13w_weeks_count,

    -- Pre 13w: 13 weeks immediately before the post window
    AVG(p.price_per_bar) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_avg_price_per_bar,
    SUM(p.base_units) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_base_units,
    AVG(p.avg_weekly_units_spm) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_velocity_spm,
    SUM(p.promo_weeks) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_promo_weeks,
    AVG(p.arp_pct_discount_any_promo) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_arp_discount,
    AVG(p.tdp) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_tdp,
    COUNT(*) FILTER (
      WHERE p.__time BETWEEN
        TIMESTAMPADD(WEEK,-25,w.__time) AND TIMESTAMPADD(WEEK,-13,w.__time)
    ) AS pre_13w_weeks_count

  FROM "price_elasticity_weekly_features" w
  JOIN "price_elasticity_weekly_features" p
    ON  p.upc           = w.upc
    AND p.channel_outlet = w.channel_outlet
    AND p.retail_account = w.retail_account
    AND p.geography_raw  = w.geography_raw
    AND p.__time BETWEEN TIMESTAMPADD(WEEK,-25,w.__time) AND w.__time
  WHERE w.parent_brand = 'BUILT'
  GROUP BY
    w.__time, w.upc, w.description, w.channel_outlet,
    w.retail_account, w.geography_raw, w.geography_level,
    w.parent_brand, w.specific_flavor_normalized,
    w.spins_flavor, w.pack_count
)

SELECT
  window_end_date                                        AS __time,
  upc, description,
  channel_outlet, retail_account,
  geography_raw, geography_level,
  parent_brand, specific_flavor_normalized,
  spins_flavor, pack_count,
  '13w'                                                  AS window_type,

  pre_13w_avg_price_per_bar, post_13w_avg_price_per_bar,
  post_13w_std_price_per_bar,
  pre_13w_base_units, post_13w_base_units,
  pre_13w_velocity_spm, post_13w_velocity_spm,
  pre_13w_promo_weeks, post_13w_promo_weeks,
  pre_13w_arp_discount, post_13w_arp_discount,
  pre_13w_tdp, post_13w_tdp,
  pre_13w_weeks_count, post_13w_weeks_count,

  -- Derived signals
  (post_13w_avg_price_per_bar - pre_13w_avg_price_per_bar)
    / NULLIF(pre_13w_avg_price_per_bar, 0)               AS price_per_bar_pct_chg,
  (post_13w_base_units - pre_13w_base_units)
    / NULLIF(pre_13w_base_units, 0)                      AS base_units_pct_chg,
  (post_13w_velocity_spm - pre_13w_velocity_spm)
    / NULLIF(pre_13w_velocity_spm, 0)                    AS velocity_spm_pct_chg,
  (post_13w_tdp - pre_13w_tdp)
    / NULLIF(pre_13w_tdp, 0)                             AS tdp_pct_chg,

  -- Promo confounded flag
  CASE
    WHEN ABS(post_13w_promo_weeks - pre_13w_promo_weeks) > 2
      OR ABS(post_13w_arp_discount - pre_13w_arp_discount) > 0.05
      THEN 1
    ELSE 0
  END AS promo_confounded,

  -- Naive elasticity estimate (Python LightGBM replaces this with a controlled value)
  CASE
    WHEN ABS((post_13w_avg_price_per_bar - pre_13w_avg_price_per_bar)
              / NULLIF(pre_13w_avg_price_per_bar, 0)) > 0.01
      THEN ((post_13w_base_units - pre_13w_base_units)
              / NULLIF(pre_13w_base_units, 0))
           / ((post_13w_avg_price_per_bar - pre_13w_avg_price_per_bar)
               / NULLIF(pre_13w_avg_price_per_bar, 0))
    ELSE NULL
  END AS naive_price_elasticity

FROM windows
WHERE
  pre_13w_weeks_count  >= 8
  AND post_13w_weeks_count >= 8
  AND pre_13w_base_units > 0
PARTITIONED BY DAY
```

## Q20 MULO FOOD pack size norms

```sql
REPLACE INTO "mulo_food_pack_size_norms"
```

## Q21 Flavor vs. protein driver features

**Purpose:** Compare flavor-group and protein-content contributions to velocity and store penetration vs. the within-flavor-group average. Run conditionally (same trigger as Q20).

```sql
REPLACE INTO "flavor_protein_driver_features"
OVERWRITE ALL
SELECT
  __time, upc, description, source_brand,
  spins_flavor_raw, source_pack_count,

    AVG(avg_weekly_units_spm) OVER (
      PARTITION BY spins_flavor_raw,
                   channel_outlet, retail_account, geography_raw, __time
    ), 0)                                               AS velocity_flavor_index,

  -- Protein tier
  CASE
    WHEN CAST(REGEXP_EXTRACT(nfp_protein_range, '([0-9]+)', 1) AS BIGINT) >= 20
      THEN 'HIGH_PROTEIN_20g_plus'
    WHEN CAST(REGEXP_EXTRACT(nfp_protein_range, '([0-9]+)', 1) AS BIGINT) >= 15
      THEN 'MID_PROTEIN_15_19g'
    WHEN CAST(REGEXP_EXTRACT(nfp_protein_range, '([0-9]+)', 1) AS BIGINT) >= 10
      THEN 'LOW_PROTEIN_10_14g'
    ELSE 'UNKNOWN'
  END AS protein_tier,

  -- Store penetration index vs same-FLAVOR-group average
  pct_stores_selling / NULLIF(
    AVG(pct_stores_selling) OVER (
      PARTITION BY spins_flavor_raw,
                   channel_outlet, retail_account, geography_raw, __time
    ), 0)                                               AS penetration_flavor_index

  subcategory = 'WELLNESS & NUTRITION BARS'
  AND channel_outlet != 'CONVENTIONAL|MILITARY'

  AND source_pack_count > 0
  AND nfp_protein IS NOT NULL
PARTITIONED BY DAY
```

## Q22 Price event queue

**Purpose:** Create the `price_event_queue` table and pre-populate the two event types that are fully deterministic from Druid data. The remaining eight event types (`DRASTIC_PRICE_CHANGE`, `PROMO_RESPONSE_BREAKPOINT`, `NEW_ITEM_PRICE_BASELINE`, `PACK_NORM_GAP`, `ELASTICITY_CONFIDENCE_DOWNGRADE`, `PRICE_DEFENSE_OPPORTUNITY`, `PRICE_DONOR_OVERLAP`, and scored variants) are written by `MO_14.7_price_events.py` after ML scoring completes.

```sql
REPLACE INTO "price_event_queue"
OVERWRITE ALL
WITH

-- COMPETITIVE_PRICE_GAP: BUILT >= 9% above Tier 1 same-FLAVOR competitor
competitive_gap_events AS (
  SELECT
    MAX(pcw.__time)                                      AS __time,
    pcw.built_upc                                        AS focal_upc,
    pcw.built_description                                AS focal_description,
    pcw.channel_outlet, pcw.retail_account,
    pcw.geography_raw                                    AS geography_display,
    pcw.geography_level,
    'COMPETITIVE_PRICE_GAP'                              AS event_type,
    'Price gap: BUILT '
      || pcw.built_description
      || ' vs '
      || MIN(pcw.competitor_brand)
      || ' — gap '
      || CAST(ROUND(AVG(pcw.price_gap_pct_vs_competitor) * 100, 1) AS VARCHAR)
      || '%'                                             AS event_label,
    'red'                                                AS event_color,
    'High'                                               AS confidence,
    AVG(pcw.price_gap_pct_vs_competitor)                 AS trigger_value,
    'price_gap_pct_vs_competitor'                        AS trigger_column,
    'price_competitive_weekly'                           AS source_table,
    '13w'                                                AS scoring_window,
    0                                                    AS cross_tool_flag,
    NULL                                                 AS cross_tool_event_id,
    CURRENT_TIMESTAMP                                    AS scored_at
  FROM "price_competitive_weekly" pcw
  WHERE
    pcw.competitive_gap_watch_flag = 1
    AND pcw.competitive_flavor_relationship IN
          ('SAME_SPINS_FLAVOR', 'SAME_CANONICAL_FLAVOR')
  GROUP BY
    pcw.built_upc, pcw.built_description,
    pcw.channel_outlet, pcw.retail_account,
    pcw.geography_raw, pcw.geography_level
  HAVING AVG(pcw.price_gap_pct_vs_competitor) >= 0.09
),

-- PACK_LADDER_COMPRESSION: 4pk→12pk per-bar gap < 10%
ladder_compression_events AS (
  SELECT
    MAX(ppl.__time)                                      AS __time,
    ppl.focal_upc,
    ppl.focal_description,
    ppl.channel_outlet, ppl.retail_account,
    ppl.geography_raw                                    AS geography_display,
    ppl.geography_level,
    'PACK_LADDER_COMPRESSION'                            AS event_type,
    'Pack ladder compression: '
      || ppl.focal_description
      || ' → '
      || MIN(ppl.partner_description)
      || ' per-bar gap below 10%'                        AS event_label,
    'blue'                                               AS event_color,
    'Medium'                                             AS confidence,
    AVG(ppl.trade_up_savings_pct)                        AS trigger_value,
    'trade_up_savings_pct'                               AS trigger_column,
    'price_pack_ladder_weekly'                           AS source_table,
    '13w'                                                AS scoring_window,
    0                                                    AS cross_tool_flag,
    NULL                                                 AS cross_tool_event_id,
    CURRENT_TIMESTAMP                                    AS scored_at
  FROM "price_pack_ladder_weekly" ppl
  WHERE
    ppl.ladder_compression_flag = 1
    AND ppl.pack_direction = 'TRADE_UP_AVAILABLE'
  GROUP BY
    ppl.focal_upc, ppl.focal_description,
    ppl.channel_outlet, ppl.retail_account,
    ppl.geography_raw, ppl.geography_level
)

SELECT * FROM competitive_gap_events
UNION ALL
SELECT * FROM ladder_compression_events
PARTITIONED BY DAY
```

**Note:** The Python script `MO_14.7_price_events.py` appends the remaining event types to this table using `INSERT INTO` after ML scoring completes. Do not overwrite the table after the Python job has run — use INSERT, not REPLACE.
